Google Sheets Setup Guide
Connect the booking form to a Google Sheet in about 10 minutes — no coding experience needed.
What this does: Every time a customer submits the booking form, their info automatically appears as a new row in your Google Sheet. You'll see their name, phone, dog details, and when they submitted — all in one place.
Setup Steps
1
Create your Google Sheet

Go to sheets.google.com and create a new blank spreadsheet. Name it something like WWW Booking Requests.

In Row 1, add these column headers exactly as shown (just type them in, one per cell starting at A1):

ColumnHeader to type
ASubmitted
BName
CPhone
DZIP
EDog Name
FBreed
GWeight
HCoat
INotes
JPhoto
2
Open Apps Script

In your Google Sheet, click the menu: Extensions → Apps Script. A new tab will open with a code editor.

Delete any existing code in the editor. Then paste in the code below:

function doPost(e) {
  var sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  var p = e.parameter;
  sheet.appendRow([
    p.submitted  || '',
    p.name       || '',
    p.phone      || '',
    p.zip        || '',
    p.dogName    || '',
    p.breed      || '',
    p.weight     || '',
    p.coat       || '',
    p.notes      || '',
    p.photoName  || ''
  ]);
  return ContentService
    .createTextOutput('OK')
    .setMimeType(ContentService.MimeType.TEXT);
}
3
Deploy as a Web App

Click Deploy → New deployment in the top-right of the Apps Script editor.

In the dialog that appears:

• Under Select type, choose Web app
• Execute as: Me
• Who has access: Anyone
• Click Deploy

Google will ask you to authorize the app. Click Authorize access, choose your Google account, and click Allow.

Important: You'll see a warning that says "Google hasn't verified this app." This is normal for your own scripts. Click Advanced → Go to [your project name] (unsafe) and then Allow.

After deploying, you'll see a Web app URL — it will look something like:
https://script.google.com/macros/s/ABC123.../exec

Copy that URL — you'll need it in the next step.

4
Add the URL to your booking page

Open customer-booking.html in a text editor (TextEdit on Mac, Notepad on Windows, or Google Chrome → right-click → View page source → copy to editor).

Near the top of the <script> section, find this line:

const SHEETS_URL = 'YOUR_APPS_SCRIPT_URL_HERE';

Replace YOUR_APPS_SCRIPT_URL_HERE with the URL you copied from Step 3. Keep the single quotes around it:

const SHEETS_URL = 'https://script.google.com/macros/s/YOUR_ACTUAL_URL/exec';

Save the file and re-upload it to your website.

5
Test it

Open your booking page in a browser and submit a test request with your own name and phone number.

Then open your Google Sheet — you should see a new row at the bottom with all the info from the form within a few seconds.

It's working! Every future booking request will automatically appear as a new row. You'll always have a running list of everyone who's asked to book — name, phone, dog info, and when they submitted.
Troubleshooting

Rows aren't appearing in the sheet
Double-check that you deployed the script with "Who has access: Anyone" — not "Only myself." If you need to change it, go back to Apps Script → Deploy → Manage deployments → edit your deployment.

I get a permissions error when deploying
Make sure you're signed into the same Google account that owns the sheet. The script has to be authorized by the account that owns the spreadsheet.

I accidentally deployed a new version
Each time you re-deploy, you get a new URL. Make sure you update the SHEETS_URL in customer-booking.html any time you redeploy.
