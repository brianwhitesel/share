Memory Layers

Two Memory Systems Working Together

You actually have two parallel memory systems. They serve different purposes and both matter.

🧠
Claude Built-in Memory
Claude.ai Settings → Memory

Your claude.ai settings have "Search and reference chats" and "Generate memory from chat history" both ON. This means Claude natively captures context from all your chats and can reference it in future sessions — even ones you start fresh.

This works automatically across the claude.ai app and web. You don't have to do anything. Updated 3 days ago per your settings screen.

📁
Cowork File Memory
~/Desktop/Brian Personal/.auto-memory/

The file-based memory system built specifically for Cowork/Dispatch. Stores structured project notes, feedback rules, and context that Claude loads at the start of every Cowork session.

More controlled and explicit than built-in memory. You can read, edit, and add to these files directly. Won't be lost if you switch accounts or devices.

Bottom line: Built-in memory catches broad patterns across all your chats automatically. File memory captures specific project detail that Cowork always has access to. Use both — they complement each other. The gap that still exists: outputs from other chats don't automatically appear in your workspace files. The Inbox drop zone is the bridge.
Automation

Running Scheduled Tasks

daily-workspace-sync
🕘 Every day at 9:05pm
Scans all of Inbox for new/unprocessed files. Processes transcripts, creates meeting summaries, flags audio needing transcription. Writes daily-sync-log.txt.
mmoc3-transcript-updater
🕘 Every day at 9:04pm
Specifically looks for MMOC3 Day 3–5 transcripts. Adds new tabs to the VIP Sessions summary page when found. Updates memory file.
Run manually first: Open the Scheduled section in the Cowork sidebar, find each task, and hit "Run now" once. This pre-approves the file access tools so future automatic runs don't pause waiting for permission.

Cross-Chat Sync

Using Other Claude Chats with This Workspace

📋
NEW CHAT CONTEXT PROMPT.txt
~/Desktop/Brian Personal/Inbox/

A ready-to-paste prompt lives in your Inbox folder. Open it, copy the whole thing, and paste it at the start of any new Claude chat (Opus, Sonnet, or any model). It gives that chat your full workspace context: folder paths, design system, active projects, memory location, and communication preferences.

That chat can then save outputs directly to your workspace folders, and the daily sync will pick them up automatically.

🔄
Workflow for any new chat
Three-step pattern

1. Paste the context prompt at the start of the new chat

2. Tell that chat to save its output to Inbox/ or the relevant subfolder

3. The 9pm sync detects the new file and processes it — or you can say "process my inbox" in Dispatch to trigger it immediately

Claude's built-in memory covers the gap too: Since "Generate memory from chat history" is ON in your settings, important things from all your chats get remembered by Claude natively — even without the Inbox trick. The two systems layer on top of each other. Built-in memory = passive and automatic. Inbox bridge = explicit and controllable.
